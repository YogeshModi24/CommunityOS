## 2. Hardened Security Controls Implemented

### 2.1 Express Gateway Bootstrapping

The Express API gateway has been hardened using industry standard security configurations:

1. **Helmet**: Applied to establish key HTTP security headers, including:
   - `Content-Security-Policy` (CSP) restrictions.
   - `Strict-Transport-Security` (HSTS) in production.
   - `X-Content-Type-Options: nosniff`.
   - `X-Frame-Options: SAMEORIGIN`.
2. **X-Powered-By**: Disabled using `app.disable('x-powered-by')` to prevent technology fingerprinting.
3. **Cookie Parser**: Integrated to support signed/secure cookie parsing and storage.
4. **Body Size Limits**: Enforced `1mb` body limits for JSON and URL-encoded requests to prevent Denial of Service (DoS) attacks via memory exhaustion.

### 2.2 Cryptographic Timing-Safe Checks

Refresh token comparisons in `AuthService` utilize timing-safe equality to protect against timing attacks. The lookup is performed on the SHA-256 hash, and value verification uses Node's native `crypto.timingSafeEqual`.

### 2.3 Route Guarding & RBAC Middlewares

1. **Rate Limiters**: Restrict brute-force attempts on sensitive endpoints:
   - `LoginLimiter`: Max 10 requests per 15 minutes.
   - `RefreshLimiter`: Max 30 requests per 15 minutes.
   - `RegisterLimiter`: Max 5 registrations per hour.
   - `PasswordLimiter`: Max 3 password resets per hour.
2. **RBAC Middleware**: Restricts access based on user role. Denied requests trigger `auth.rbac.denied` metric counters and log correlation context.
